Regra da Cascata

O CSS segue a lógica da cascata: primeiro importância, depois especificidade, e por último a ordem do código.

Especificidade

Seletor por tag tem menos peso, classe tem peso médio, e id tem o maior peso.

  • p {} → menor prioridade
  • .classe {} → prioridade média
  • #id {} → prioridade alta

Combinação de Seletores

.primeiro h1 → seleciona qualquer h1 dentro de .primeiro.

.primeira > h1 → seleciona apenas h1 filho direto de .primeira.

.primeira h1 + h1 → seleciona h1 que vem logo depois de outro h1 dentro de .primeira.

Pseudo-classes

:first-child → seleciona o primeiro filho.

:last-child → seleciona o último filho.

:nth-child(n) → seleciona o filho na posição n.

:nth-of-type(n) → seleciona o n-ésimo filho do mesmo tipo.

Display Inline vs Block

Elementos inline ocupam apenas o conteúdo e não quebram linha. Já os block ocupam a largura toda e quebram linha.

Elementos Inline e Block

  • Inline: a, span, strong, em, img
  • Block: div, p, h1-h6, section, header, footer, nav

Margin, Padding e Border

Margin: espaçamento externo.

Padding: espaçamento interno.

Border: contorno do elemento.

Shorthand:

  • 1 valor → todos os lados
  • 2 valores → cima/baixo e lados
  • 3 valores → cima, lados, baixo
  • 4 valores → cima, direita, baixo, esquerda
